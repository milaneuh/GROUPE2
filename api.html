L'API? Késaco?

Définition du mot API

En informatique, API est l'acronyme d'Application Programming Interface (interface de programmation applicative en français). C'est un ensemble de définitions et de protocoles qui facilite la création de logiciels d'applications.

Comment ça marche?

Les API permettent à votre produit ou service de communiquer avec d'autres programmes sans avoir à se soucier du fonctionnement complexe de celui-ci. Les API peuvent par exemple être utilisées pour déclencher des campagnes publicitaires d'emailing sans avoir à passer par la compréhension d'une telle application. (API AdWords de Google par exemple)

Types d'APIs

Il existe plusieurs types d'API

API privé. Ces interfaces de programmation applicative sont concus pour améliorer les solutions et les services au seins d'une entreprise. Cela permet aux développeur de l'entreprise ou aux prestataires, de intégrer des applications ou des systèmes spécifiques à l'entreprise par le biais de l'API.

API de Partenaire. Ces interfaces de programmation applicative sont promus ouvertement mais sont accecible uniquement par contrat avec le distributeur de celle-ci. L'entreprise distrubuant l'API permet à ses partenaire de avoir accès à ses données ou avoir des revenus supplémentaire grâce au flux d'utilisateurs. Mais elle peut gérer la façon d'ont son API est utilisée et faire en sorte que l'expérience utilisateurs soit décente et soit en accord avec l'identité de l'entreprise.

API Publique. Ces API sont disponible pour tous les développeurs. Elle permettent à l'entreprise d'avoir plus de recognition, voir une source de revenue en plus si l'API est exécuter correctement.
Il existe deux types d'API publiques:
Les API ouvertes (Gratuite) et les API commerciales. La définition des API ouvertes suggeste que toutes les fonctionnalitées soit gratuite et peuvent être utilisées sans restriction. Par exemple, il est possible de créer une application qui utilise une API sans avoir une approbation explicite de l'entreprise délivrant l'API. La description suggeste aussi que toutes les documentations des API ouvertes soit accecible gratuitement et que l'API peut être utilisée gratuitement pour tester ou créer des applications.
Les utilisateurs d'API commerciales paient un abonnement ou ont payés pour l'utiliser. Une approche populaire auprès des distributeur d'API et de faire des essais gratuit d'application. Comme ça les utilisateurs se font une idées avant de payer l'abonnement.
